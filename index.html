🧪 Labs

Lab 1

Basic UI Components

Building fundamental Android UI elements using Jetpack Compose — buttons, text fields, and layouts.

In this lab I learned how to build UI elements using Jetpack Compose declarative syntax. I found the difference between Column and Row challenging at first but eventually understood the layout model. This lab gave me a strong foundation for the labs ahead."

Lab 2

Interaction & State

Implementing user interactions with state management using remember and mutableStateOf.

In this lab I learned how to manage user interactions and application state using remember and mutableStateOf in Jetpack Compose. Understanding the concept of recomposition was initially confusing, but once I saw how the UI automatically updates when state changes, everything clicked. This lab shifted my thinking from static layouts to truly reactive interfaces.

Lab 3

Material Design

Applying Material Design 3 theming, colours, typography, and components to create polished UI.

This lab introduced me to Material Design 3 and how to apply consistent theming, typography, and colour schemes across an Android app. I found it rewarding to see how a few theme changes could make an app look significantly more polished and professional. It also made me appreciate the importance of visual consistency in user experience.

Lab 4

Navigation

Implementing multi-screen navigation with Jetpack Navigation Compose and passing arguments between screens.

Implementing multi-screen navigation with Jetpack Navigation Compose taught me how real apps are structured beyond a single screen. Passing arguments between destinations was tricky at first, but working through it gave me a much clearer picture of how screen flows are managed. This lab directly prepared me for building the project that followed.

Lab 5

Room Database

Persisting data locally with Room, DAO, and ViewModel following the MVVM architecture pattern.

In this lab I learned how to persist data locally using Room, including setting up entities, DAOs, and a database instance following the MVVM pattern. Connecting the database to a ViewModel and observing the data as a Flow felt complex initially, but it showed me how clean architecture keeps UI and data logic properly separated. This lab was the most technically demanding so far and the most rewarding to complete.

Project 1 · Lab 4 + Working Features

[Tripping — PROJECT 1]

Many travellers struggle to keep their trip details organised in one place, often relying on scattered notes or memory to track destinations, dates, and itinerary information. This app provides a straightforward trip management solution that allows users to record, view, and manage their travel plans entirely on-device.

Building Project 1 brought together everything I had practised across Labs 1 to 4 into a single cohesive application for the first time. 💭 Reflection & Learning Journey

Over the course of this semester, my skills in mobile application development grew considerably. What began as building simple static screens in Lab 1 evolved into creating fully interactive apps with Material Design styling, multi-screen navigation, ViewModel-based state management, and local data persistence through Room Database.

Working through each lab in sequence helped me see how the individual pieces of Android development fit together as a whole. Completing the Quick Move application in particular bridged the gap between classroom exercises and real-world problem-solving, pushing me to think like a developer rather than just a student.

Project 2 opened up a new dimension of learning by introducing technologies I had not previously worked with, including Firebase Firestore for cloud storage, a Weather API for live data, and GPS sensors for location awareness. Connecting these components taught me how contemporary mobile apps interact with external services and the physical capabilities of a device.

Looking back, this course has left me with more than just technical knowledge — it has built my confidence in tackling complex development challenges and given me a solid, end-to-end understanding of how to design, build, test, and ship a modern Android application.
